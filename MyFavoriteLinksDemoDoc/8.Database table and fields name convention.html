My database table & column name conventions

  • Database Tables & Columns: snake_case(eg. user_id, task_user)
  • Each table has a primary key column named id(automatically increase)
  • One to many foreign key : Singular table name(snake_case) _id, eg. user_id, link_id, category_id
  • Many to many tables & columns: three database tables are needed: users, roles, and role_user. The role_user table is derived from the alphabetical order of the related model names, and contains the user_id and role_id columns.
  • Date columns: use timestamps format
  • Ref: https://laravel.com/docs/7.x/eloquent#eloquent-model-conventions
  • Ref: https://laravel.com/docs/7.x/eloquent-relationships

File & class & method name conventions

  • Procedural function filename: camelCase
  • OOP Classes,Controllers filename: PascalCase, be the same as Class name
  • Template filename: snake_case(eg. new_user.tpl.php)
  • Methods,Properties,Functions,Variables,formInputName: camelCase
